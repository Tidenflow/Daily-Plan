## 260322 - TypeScript 单文件实战训练设计书

### 训练主题

实现一个 `Student Task Manager`（学生任务管理器），要求全部代码集中在一个文件中（建议：`training-260322.ts`）。

### 训练目标（基于已学知识）

1. 熟练使用基础类型、联合类型、字面量类型、枚举
2. 熟练使用接口（对象接口、函数接口、索引签名）
3. 使用函数重载、默认参数、`this` 参数、`call/apply/bind`
4. 使用类实现封装（`public/private/readonly/static`）
5. 使用继承与方法重写（`extends/super`）
6. 使用抽象类 + 多态设计统一行为
7. 使用泛型函数与泛型接口提升复用性

---

## 一、业务背景

你要做一个“学生个人任务系统”：
- 可以添加任务、更新任务状态、删除任务、查询任务
- 支持不同任务类型（普通任务、考试任务）
- 支持按条件过滤（状态/优先级/关键字）
- 支持统计（总数、完成数、按优先级分组）

---

## 二、单文件结构建议

按这个顺序写在同一个 `.ts` 文件：

1. 类型与枚举定义
2. 接口定义
3. 抽象类 + 子类
4. 泛型工具函数
5. 仓库类（任务管理核心）
6. 测试运行区（`main()`）

---

## 三、必须实现的类型与接口

### 1) 枚举与字面量

- `enum TaskStatus { TODO, IN_PROGRESS, DONE }`
- `type Priority = "low" | "medium" | "high"`

### 2) 基础接口

- `interface ITaskBase`
  - `readonly id: number`
  - `title: string`
  - `status: TaskStatus`
  - `priority: Priority`
  - `tags?: string[]`

- `interface IDescribable`
  - `getSummary(this: IDescribable): string`

- `interface IDictionary`
  - 索引签名：`[key: string]: string | number`

### 3) 泛型接口

- `interface IRepository<T>`
  - `add(item: T): void`
  - `removeById(id: number): boolean`
  - `getById(id: number): T | undefined`
  - `getAll(): T[]`

---

## 四、必须实现的类

### 1) 抽象类：`AbstractTask`

字段：
- `public title`
- `public status`
- `public priority`
- `public tags?`
- `readonly id`

行为：
- 构造函数初始化通用字段
- `abstract getSummary(): string`
- `markDone(): void`（具体方法）

### 2) 子类一：`NormalTask extends AbstractTask`

- 新增字段：`deadline?: Date`
- 重写 `getSummary()`

### 3) 子类二：`ExamTask extends AbstractTask`

- 新增字段：`subject: string`, `score?: number`
- 重写 `getSummary()`

### 4) 核心管理类：`TaskManager implements IRepository<AbstractTask>`

要求：
- `private tasks: AbstractTask[] = []`
- `private static nextId = 1`
- `createId(): number`（可私有）
- 实现仓库接口全部方法
- 提供扩展方法：
  - `updateStatus(id: number, status: TaskStatus): boolean`
  - `filterByStatus(status: TaskStatus): AbstractTask[]`
  - `filterByPriority(priority: Priority): AbstractTask[]`
  - `searchByKeyword(keyword: string): AbstractTask[]`
  - `getStats(): { total: number; done: number; todo: number }`

---

## 五、必须实现的函数（体现函数专题）

### 1) 函数重载

实现 `formatTask`：
- 传 `AbstractTask` 返回 `string`
- 传 `AbstractTask[]` 返回 `string[]`

### 2) 默认参数

实现 `printTasks(tasks: AbstractTask[], showIndex: boolean = true): void`

### 3) `this` + call/apply/bind

实现：
- `function logTask(this: { prefix: string }, task: AbstractTask): void`
- 用 `call` / `apply` / `bind` 分别调用一次

---

## 六、必须实现的泛型函数

1. `pickField<T, K extends keyof T>(obj: T, key: K): T[K]`
2. `makePair<S, T>(first: S, second: T): [S, T]`
3. `identity<T>(value: T): T`

---

## 七、运行演示要求（main函数）

`main()` 至少完成：

1. 创建 `TaskManager`
2. 添加至少 5 个任务（`NormalTask` 和 `ExamTask` 混合）
3. 更新 2 个任务状态
4. 打印全部任务摘要
5. 执行状态过滤、优先级过滤、关键字搜索
6. 打印统计信息
7. 演示函数重载与 call/apply/bind
8. 演示泛型函数调用结果

---

## 八、验收标准

1. 代码在单文件内可直接运行（`ts-node` 或编译后 `node`）
2. 无 `any` 滥用（必要时优先 `unknown` + 类型收窄）
3. 抽象类、继承、泛型、接口、重载都真实使用到
4. 输出日志能明显看出各功能工作正常
5. 关键方法具备基础边界处理（如 id 不存在返回 `false`）

---

## 九、加分挑战（可选）

1. 增加 `BatchTask<T extends AbstractTask>` 泛型容器类
2. 增加 `readonly createdAt`
3. 增加任务排序：按优先级 + 截止日期
4. 用 `never` 做穷尽检查（如处理状态分支）

2. 写抽象类和两个子类（15分钟）
