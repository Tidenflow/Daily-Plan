## CWFrame 明日目标（Day 1）

### 阶段一：基础设施

- 目录结构：`routes/ services/ repos/ data/`
- 统一错误返回格式
- `.env` + 端口配置
- 搭建后端骨架（Express + TS）
- 跑通健康检查接口：`GET /api/health`

### 阶段二：用户系统（M2 核心）

- **注册** `POST /api/auth/register`
  - 用户名重复校验
- **登录** `POST /api/auth/login`
  - 用户名 + 密码校验（先明文，后续再升级）
  - 返回 userId
- 存储到 `data/users.json`

### 阶段三：进度系统（M2 核心）

- `GET /api/progress/:userId` 读取进度
- `PUT /api/progress/:userId` 更新进度
- 存储到 `data/progress.json`

### 阶段四：前后端联调

- 前端 `cwframe.loader` 改为请求 API
- 完成"登录后读取进度、解锁后保存进度"
